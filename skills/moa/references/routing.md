# auto 模式路由决策表（仲裁人执行）

auto 模式下，你（当前 agent）在召集委员前做三步判断，并把结论**一句话公示**给用户，例如：
> 场景=决策，难度=L2，阶段=架构 → 3 委员（A 逻辑 / C 事实 / D 反方），流水线=认领→交叉审查→仲裁，预估 API 计费 3 席。

路由是 agent 侧逻辑（moa.py 不做路由）；你据本表构造 config 后调用 moa.py。

## Step 0：是否该启动（L0 双闸门）

- 简单问答、答案基本唯一 → **不启动**，直接回答。
- **可机械验证的客观问题**（算术、事实检索、"这段代码输出什么"）→ **不启动**，直接跑验证/执行。MoA 对此类任务实测收益为负（togethercomputer/moa issue #41）。
- 判断类、错误代价高、有多个合理答案 → 进 Step 1。

## Step 1：场景识别

| 信号 | 场景 | mode |
|---|---|---|
| 评审/审查/审核/把关/review 一份方案、代码、文档 | 评审 | review |
| 多个选项/方向/答案取舍，"选 A 还是 B" | 决策 | decide |
| 想点子/差异化/发散 | 头脑风暴 | brainstorm |
| "对上面的分析/结果二次确认" | 二次确认 | review（独立回答流水线） |
| "对上面的总结做出评审/建议" | 总结评审 | review |

多场景复合（如"评审现有方案并提替代方案"）→ 拆两次调用：先 review 后 brainstorm。

## Step 2：难度评估 → 委员数 + 精炼轮

判据：错误代价（可逆性）、分歧空间（合理答案是否多个）、材料复杂度。

| 难度 | 定义 | 委员数 | refine-rounds |
|---|---|---|---|
| L1 | 局部、低风险、可逆判断 | 1–2 | 0 |
| L2 | 常规方案评审、普通技术选型 | 3 | 0–1 |
| L3 | 架构决策、不可逆选择、交付仲裁、安全审计 | 4（可 D 席拆 D1+D2，总调用 ≤5） | 1–2（**根本分歧未化解 + 用户明确要"真辩一轮" → 精炼换开会讨论**，见下方 L3 选路门） |

> 拆 D1+D2 时:seat `D1`/`D2` 在 `DEFAULT_SEAT_ROLE`(moa.py)**无默认角色映射**,会回退到通用兜底 prompt。必须给这两席显式设 `member.role`(或 `custom_roles`),否则两个红队席会拿到一样的泛化提示、失去分化意义。

## Step 3：阶段识别 → 激活哪些席位

| 阶段 | 内容 | 默认激活席位 |
|---|---|---|
| 架构 | 需求/方案设计，对文档/计划/头脑风暴评审 | A + D + C |
| 编码 | 实现期 bug/问题/功能分析建议 | B + A |
| 测试 | 真实流程验证、测试问题多人复核 | B + D |
| 审查 | 全项目查缺陷/安全/技术债 | B + C + 安全角色 |
| 仲裁 | 是否达交付标准、多方案准确建议 | A + C + D（全员亦可） |

## Step 4：模型分配 → 通道落地

席位确定后，按通道可用性把 A/B/C/D 落到具体模型（config members）：
- **优先订阅通道**（省钱）：Claude 系 → `channel: subagent`（你用 Task 派发，可指定非会话默认模型）；GPT 系 → `channel: cli`（codex）。
- **API 补长尾**：Gemini/Grok 及其他 → `channel: api`（openrouter）。
- 每席配 `fallback` 降级链（订阅通道失败 → api）。
- 仲裁人永不降级：fallback 只降委员，你（仲裁人）始终是当前最强 agent。

## 场景 → 流水线映射（互动模式组装）

| 场景 | 生成 | 精炼（L2+） | 收敛 |
|---|---|---|---|
| 评审/审查/审核 | 角色扮演（roles-review） | 匿名互评 | 主席综合 |
| 决策/推荐 | 角色扮演（认领选项+对抗论证，roles-decide） | 交叉审查 | 仲裁（对比矩阵+RECOMMEND+失效条件） |
| 头脑风暴 | 角色扮演（发散人格，高 temp） | 无 | 策展（孤例保护） |
| 二次确认 | 独立回答 | 无 | 主席综合（共识/分歧） |
| 总结评审 | 独立回答或角色扮演 | L3：匿名互评 | 主席综合 |

## 决策场景的选项认领规则

- 选项数 > 席位数 → 按重要度合并相近/弱选项后取前 N，候选中**必须含"维持现状"**。
- 席位数 > 选项数 → 余席改任反方或事实核查。

## 开会讨论的 L3 选路门（§6 阶段5;auto 只在满足全部条件时才提议）

开会讨论成本最高、从众风险最高,**auto 默认不选它**。仅当**三个条件同时成立**时,auto 才在公示里把它作为精炼阶段的候选提出:

1. **难度 = L3**（架构/不可逆/交付仲裁/安全审计）——L1/L2 一律不考虑;
2. **根本分歧未化解**——已有生成轮(或一轮匿名互评/交叉审查)显示委员在核心结论上对立且各执证据,不是措辞差异;
3. **用户明确要求**——用户说了"让它们真辩一轮 / 开会讨论 / 多轮辩论"之类。第 2 条满足但用户没提,auto **不自动升级**,仍走交叉审查/匿名互评。

三条都成立 → auto 一句话公示时提议:`难度=L3、根本分歧未化解、你要求真辩 → 精炼阶段用开会讨论(顺序发言,成本最高),预估 N 席 × M 轮`,**经用户确认后**再按 `references/discuss.md` 逐回合编排。任一条不成立 → 精炼默认交叉审查(决策)/匿名互评(评审)。

> 判定优先级高于"给结论的冲动":第 3 条是硬门。用户没显式要,哪怕分歧再大也不擅自开会讨论——那是成本与从众风险的主动选择,决定权在用户。

## Self-MoA 两条路径

- **主动**：custom 模式复制同一模型成多席 → 为重复席位自动分配互不相同的角色，报告注明 Self-MoA 构成。适合只有一个强通道但想要多视角时。命令二选一：单模型 + `--members N`（复制成 N 席，如 `--members 3 --models "openai/gpt-5.6-sol"`），或 `--models` 里显式重复列出同一模型 ID。**无 `--self-moa` 这个标志**（脚本只有 `--members`/`--models`）。
- **兜底**：无任何外部通道可用 → 你分回合独立扮演各角色（每回合只依据简报+角色契约，刻意不参考此前回合产出），产出写独立文件后统一收敛。**必须声明**：只有角色分化收益，无跨模型错误去相关收益。
